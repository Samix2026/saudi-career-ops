# Contributing to saudi-career-ops

شكراً لاهتمامك بالمساهمة. هذا المشروع يعتمد على خبرة المجتمع لتغطية عمق السوق السعودي — لا يمكن لشخص واحد أن يعرف كل قطاع وكل منطقة.

> English speakers: this guide is bilingual. All contribution types are open to everyone regardless of language.

---

## قبل أي شيء — اقرأ هذا

المشروع **prompt-first** — قيمته في ملفات `modes/` و`_shared.md`، ليس في كود معقد. هذا يعني أن المساهمة ممكنة لأي شخص يعرف السوق السعودي جيداً، حتى لو لم يكتب كوداً من قبل.

أبسط مساهمة ممكنة: **افتح Issue وأخبرنا أين أخطأ النظام**.

---

## أنواع المساهمات

### سهل — نقطة بداية جيدة

**تصحيح بيانات `_shared.md`**
إذا رأيت رقم راتب غير دقيق، اسم شركة خاطئ، أو معلومة نطاقات متقادمة — افتح PR مباشرةً مع المصدر.

**إضافة شركة لقائمة PIF entities أو RHQ**
القائمة الحالية غير مكتملة. أضف شركة مع: الاسم، الموقع، رابط careers، القطاع.

**اختبار mode على وظيفة حقيقية**
شغّل `/saudi-career-ops وظيفة` على وظيفة حقيقية من السوق وأخبرنا:
- ما الذي أصاب النظام؟
- ما الذي أخطأ أو أغفل؟
- هل Block B (النطاق) كان دقيقاً؟

افتح Issue بعنوان: `[test] [القطاع] — [ملاحظاتك]`

---

### متوسط — يحتاج معرفة بالسوق

**إضافة بيانات قطاع متخصص**

القطاعات التالية تحتاج تعمقاً أكبر في `_shared.md`:

| القطاع | ما ينقص |
|--------|---------|
| الرعاية الصحية | نسب نطاقات محدّثة، رواتب التخصصات الصحية بالمدن |
| الهندسة | متطلبات اعتماد هيئة المهندسين بالتفصيل |
| التعليم | وظائف القطاع التعليمي الخاص والحكومي، نسب السعودة |
| السياحة والضيافة | قطاع جديد بنسب نطاقات متغيرة منذ 2025 |
| اللوجستيات | NEOM logistics، ميناء جدة، المدينة الصناعية |
| التقنية المالية | بيانات رواتب fintech السعودي، مقارنة البنوك الرقمية |

افتح Issue أولاً تصف ما ستضيف، ثم PR.

**تحسين mode قائم**

إذا وجدت أن أحد الـ modes يُنتج إجابات عامة أو غير دقيقة لقطاعك:
1. افتح Issue تصف المشكلة بمثال محدد
2. اقترح التعديل بالنص
3. سنناقش ونُدمج

---

### متقدم — يحتاج تقنية

**إضافة mode جديد**

قبل البدء: افتح Issue بعنوان `[mode proposal] اسم-الـ-mode` وصف فيها:
- ما المشكلة التي يحلها؟
- كيف يختلف عن الـ modes الموجودة؟
- ما هيكل المخرج المقترح؟

انتظر موافقة قبل الكتابة — لتجنب التكرار أو التعارض.

هيكل كل mode جديد:
```markdown
# اسم الـ mode — وصف قصير

## الغرض
## كيفية الاستخدام
## تعليمات الـ Agent
## هيكل التقرير (blocks أو sections)
## ما لا يفعله هذا الـ mode
```

**Phase 3 — البنية التقنية**

إذا كنت مطوراً وتريد المساهمة في:
- `templates/cv-template-ar.html` — قالب CV عربي RTL + Cairo font
- Playwright scanner للبوابات السعودية
- Go dashboard
- GitHub Actions CI

افتح Issue بعنوان `[phase3] ما تريد بناءه` للتنسيق مع باقي المساهمين.

## كيف تفتح PR

```bash
# 1. Fork الريبو
# 2. أنشئ branch باسم وصفي
git checkout -b fix/nitaqat-healthcare-rates
# أو
git checkout -b add/eastern-province-salary-data
# أو
git checkout -b mode/دفعة

# 3. عدّل الملفات
# 4. Commit بعنوان واضح
git commit -m "fix: update healthcare Nitaqat rates to April 2026 changes"
# أو
git commit -m "data: add Eastern Province salary benchmarks for engineering roles"

# 5. افتح PR مع وصف يشمل:
#    - ما الذي غيّرته؟
#    - لماذا؟
#    - المصدر إذا كانت بيانات
```

## معايير الجودة

**للبيانات والأرقام:**
- أذكر المصدر دائماً — Bayt Salary Survey، LinkedIn Salary SA، تقرير وزارة الموارد البشرية
- أذكر السنة — البيانات القديمة أسوأ من غياب البيانات
- لا تعتمد على الذاكرة لأرقام النطاقات — تحقق من قيوة أو HRSD

**للـ modes:**
- كل تعليمة للـ agent يجب أن تكون محددة وقابلة للتنفيذ
- لا تكرر ما في `_shared.md` — أشر إليه
- كل mode يُنهي بقسم "ما لا يفعله هذا الـ mode"
- اختبر الـ mode على وظيفة حقيقية قبل فتح PR

**للغة:**
- العربية: بدون تشكيل، بدون إنجليزي مُعرَّب داخل الجمل
- الإنجليزي: sentence case، مباشر، بدون مصطلحات عربية غير مشروحة
