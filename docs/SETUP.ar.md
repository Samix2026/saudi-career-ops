# دليل الإعداد — saudi-career-ops

> من الصفر إلى أول تقييم وظيفة في أقل من 30 دقيقة.

---

## المتطلبات

شيء واحد فقط مطلوب:

- **Claude Code** — مثبّت ومُهيَّأ على جهازك

لا npm، لا Python، لا إعداد خوادم. النظام يعمل بملفات Markdown وClaude Code فقط.

> إذا لم يكن Claude Code مثبتاً: [تعليمات التثبيت](https://docs.anthropic.com/en/docs/claude-code)

## الإعداد — 4 خطوات

---

### الخطوة 1 — استنسخ المشروع

```bash
git clone https://github.com/Samix2026/saudi-career-ops.git
cd saudi-career-ops
```

---

### الخطوة 2 — أنشئ ملف السيرة الذاتية

هذا أهم ملف في النظام. كل تقييم وكل تحليل يبدأ منه.

```bash
# أنشئ الملف
touch cv.md
```

افتح `cv.md` وأضف سيرتك الذاتية بهذا الهيكل:

```markdown
# [اسمك الكامل]

## المعلومات الأساسية
- المسمى الحالي: [مسماك الوظيفي]
- القطاع: [قطاعك]
- سنوات الخبرة: [X سنة]
- الموقع: [مدينتك]
- الجنسية: [سعودي / مقيم + الجنسية]

## الخبرات المهنية

### [المسمى الوظيفي] — [اسم المنشأة] | [من — إلى]
- [إنجاز محدد بالأرقام: حققت X مما أدى إلى Y]
- [إنجاز ثانٍ]
- [إنجاز ثالث]

### [المسمى السابق] — [اسم المنشأة] | [من — إلى]
- [إنجاز محدد]

## المهارات الجوهرية
[قائمة بأبرز مهاراتك التقنية والإدارية]

## المؤهلات والشهادات
- [الدرجة العلمية] — [الجامعة] — [السنة]
- [شهادة مهنية] — [الجهة المانحة] — [السنة]

## اللغات
- العربية: لغة أم
- الإنجليزية: [مستواك]
```

**نصائح لكتابة cv.md الفعّال:**

كل إنجاز يجب أن يحتوي على **رقم** — النسبة أو المبلغ أو الوقت. "حسّنت العمليات" لا قيمة له. "قلّصت وقت المعالجة من 5 أيام إلى يوم واحد" هو ما يبني قصة STAR.

لا تنسخ سيرتك من PDF — اكتبها بصيغة سردية مُركَّزة على النتائج.

`cv.md` في `.gitignore` — لن يُرفع على GitHub أبداً. بياناتك تبقى على جهازك.

### الخطوة 3 — عبّئ ملف الإعدادات الشخصية

```bash
cp config/profile.example.yml config/profile.yml
```

افتح `config/profile.yml` وعبّئ هذه الحقول كحد أدنى:

```yaml
# الهوية المهنية
name: "[اسمك]"
current_title: "[مسماك الحالي]"
nationality: "[سعودي | مقيم]"
location: "[الرياض | جدة | الدمام | NEOM | أخرى]"

# ما تبحث عنه
target_title: "[المسمى الوظيفي المستهدف]"
target_sectors:
  - "[القطاع الأول]"
  - "[القطاع الثاني]"
target_org_types:
  - "[PIF entity | خاصة | RHQ | حكومية]"

# الراتب
salary_floor_sar: [الحد الأدنى المقبول بالريال — رقم فقط]
salary_target_sar: [الراتب المستهدف]

# مستوى الخبرة
seniority: "[مبتدئ | متوسط | أقدم | مدير | قيادي]"
years_experience: [عدد السنوات]
```

### الخطوة 4 — اختبر الإعداد

افتح Claude Code في مجلد المشروع:

```bash
claude
```

ثم اكتب:

```
/saudi-career-ops واقع
```

إذا رأيت تقريراً يذكر اسمك ومسماك وقطاعك — الإعداد يعمل.

## أول تقييم وظيفة

الآن النظام جاهز. هذا هو الـ workflow الكامل:

**الخطوة أ — ابحث عن وظيفة:**
```
/saudi-career-ops مسح
```
احصل على خطة بحث مُخصَّصة لكل بوابة سعودية.

**الخطوة ب — تحقق من الأهلية (30 ثانية):**
```
/saudi-career-ops نطاق [اسم المنشأة] [الدور]
```
هل الدور متاح لك تنظيمياً؟

**الخطوة ج — التقييم الكامل:**
```
/saudi-career-ops وظيفة
[الصق الوصف الوظيفي]
```
7 blocks كاملة: مطابقة السيرة، النطاق، الراتب، إعداد المقابلة، الحكم النهائي.

**الخطوة د — إذا حصلت على موعد مقابلة:**
```
/saudi-career-ops تحضير [اسم المنشأة] [المسمى]
```

## الأوامر الكاملة

| الأمر | ما يفعله |
|-------|---------|
| `/saudi-career-ops مسح` | خطة بحث مُخصَّصة لكل بوابة سعودية |
| `/saudi-career-ops نطاق [منشأة] [دور]` | تحليل النطاق وأهليتك في دقيقتين |

| `/saudi-career-ops تواصل [نوع] [سياق]` | رسالة LinkedIn جاهزة |

## أسئلة شائعة

**هل أحتاج اشتراكاً مدفوعاً؟**
Claude Code يحتاج اشتراك Anthropic. لا توجد تكاليف إضافية للمشروع نفسه.

**هل بياناتي آمنة؟**
`cv.md` و`reports/` و`data/` كلها في `.gitignore`. لا شيء يُرفع على GitHub. بياناتك تُرسل فقط لـ Anthropic API عند تشغيل الأوامر — نفس ما يحدث عند استخدام Claude Code في أي مشروع.

**هل يعمل مع وظائف بالعربية؟**
نعم. الـ modes تدعم الوصف الوظيفي بالعربية والإنجليزي وتُنتج التقرير بلغة المدخل.

**كيف أُحدّث المشروع؟**
```bash
git pull origin main
```
ملفاتك الشخصية (`cv.md`، `profile.yml`، `data/`، `reports/`) لا تتأثر بالتحديثات.

**وجدت خطأ أو لديّ اقتراح؟**
افتح [Issue على GitHub](https://github.com/Samix2026/saudi-career-ops/issues) — راجع `CONTRIBUTING.md` لتعرف كيف تساهم.
